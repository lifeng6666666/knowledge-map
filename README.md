# 知识地图 (Knowledge Map)

一个可视化的知识图谱工具，支持在浏览器中创建、编辑和管理课程与概念的关系网。

**现在包含：34 个节点 | 41 条关系 | 4 大学科体系 | 50+ 推荐课程资源**

## 功能特性

- 📊 **网状图可视化**：使用 vis.js 绘制动态网络图
- ✏️ **图形化编辑**：无需代码，直接在界面上添加节点和关系
- 📚 **丰富的节点信息**：支持课程描述、推荐教科书和推荐课程
- 🔗 **灵活的关系类型**：前置课程、后继课程、相关课程、应用关系
- 💾 **数据管理**：
  - 本地存储（自动保存）
  - JSON 导入/导出
  - 下载为 JSON 文件
- 🌐 **GitHub Pages 部署**：直接发布到 GitHub Pages
- 🎨 **美观的 UI**：深色侧边栏 + 网络可视化区域

## 快速开始

### 方式 1：本地使用

1. **克隆仓库**
   ```bash
   git clone https://github.com/yourusername/knowledge-map.git
   cd knowledge-map
   ```

2. **打开在浏览器**
   - 直接打开 `index.html`，或
   - 使用本地服务器：`python -m http.server 8000`
   - 访问 `http://localhost:8000`

### 方式 2：GitHub Pages 部署

1. **创建 GitHub 仓库**
   - 在 GitHub 上创建名为 `knowledge-map` 的仓库
   - 将代码推送到 `main` 分支

2. **启用 GitHub Pages**
   - 进入仓库 Settings → Pages
   - 选择 Branch: `main` / Folder: `/ (root)`
   - 点击 Save

3. **访问网站**
   - URL: `https://yourusername.github.io/knowledge-map/`

## 使用指南

### 快速导入示例数据

打开应用后，你会看到预设的示例数据已加载，包含：
- 完整的计算机科学学习路径
- 数学和经济学基础课程
- 软技能和实践活动

如果你想使用完整的知识地图数据：
1. 点击左侧 "数据" 标签
2. 点击 "⬆️ 导入 JSON"
3. 打开 `knowledge-map-complete.json` 文件
4. 复制全部内容到对话框
5. 点击 "导入"

### 了解数据结构

详见 **KNOWLEDGE_MAP_STRUCTURE.md** 文件，包含：
- 完整的节点和关系说明
- 学习路径示例
- 如何扩展知识地图

### 添加节点

1. 在左侧面板选择节点类型：📖 课程 或 🎯 概念
2. 填写节点名称（必填）
3. 可选：添加描述、教科书、推荐课程
4. 点击 "✨ 添加节点"

**节点类型**：
- **课程节点**（蓝色正方形）：代表具体的课程
- **概念节点**（金色菱形）：代表学科领域或概念，可关联多个课程

### 添加关系

1. 选择 "从节点" 和 "到节点"
2. 选择关系类型：
   - **前置课程** 🔴：学习当前课程的前提
   - **后继课程** 🟢：接续的高级课程
   - **相关课程** 💚：相关但无严格顺序
   - **应用** 💛：知识的应用领域
3. 点击 "🔗 添加关系"

### 编辑和管理

- **查看详情**：点击任意节点，右侧面板显示详情
- **删除节点**：选中节点，点击"🗑️ 删除选中节点"
- **交互操作**：
  - 拖拽节点移动
  - 滚轮缩放
  - 右键拖拽平移

### 数据管理

**导出数据**：
- 点击"数据"标签页
- 点击"⬇️ 导出 JSON"

**导入数据**：
- 点击"⬆️ 导入 JSON"
- 粘贴 JSON 数据
- 点击"导入"

**自动保存**：
- 所有更改自动保存到浏览器本地存储
- 刷新页面数据不会丢失

## JSON 数据格式

```json
{
  "nodes": [
    {
      "id": "node_1",
      "label": "微观经济学原理",
      "type": "course",
      "description": "研究个体经济主体...",
      "books": "教科书1\n教科书2",
      "courses": "推荐课程1\n推荐课程2"
    }
  ],
  "edges": [
    {
      "id": "node_1_node_2",
      "from": "node_1",
      "to": "node_2",
      "label": "前置",
      "type": "prerequisite"
    }
  ]
}
```

## 常见问题

**Q: 数据会丢失吗？**
A: 不会。所有修改自动保存到浏览器的 localStorage。你也可以定期导出 JSON 备份。

**Q: 如何与他人分享？**
A: 
1. 导出数据为 JSON
2. 发送 JSON 文件给他人
3. 他人通过"导入 JSON"功能导入

**Q: 能自定义节点样式吗？**
A: 可以。编辑 `app.js` 中的颜色配置部分修改节点和边的样式。

**Q: 能导出为图片吗？**
A: 可以使用浏览器的截图功能或按 F12 打开开发者工具截图。
