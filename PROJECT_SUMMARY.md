## 🎯 核心功能详解

### 1. 节点编辑面板
**位置**：左侧边栏 → "编辑" 标签

**支持的操作**：
```
添加节点：
├─ 选择类型（课程/概念）
├─ 输入名称 [必需]
├─ 添加描述 [可选]
├─ 列出教科书 [可选]
├─ 列出推荐课程 [可选]
└─ 点击添加

添加关系：
├─ 选择 "从节点"
├─ 选择 "到节点"
├─ 选择关系类型
│  ├─ 前置课程 (前提条件)
│  ├─ 后继课程 (进阶课程)
│  ├─ 相关课程 (相关知识)
│  └─ 应用 (应用领域)
└─ 点击添加

操作：
├─ 删除选中节点 (自动删除相关关系)
└─ 清空所有 (谨慎操作)
```

### 2. 节点详情面板
**位置**：左侧边栏 → "详情" 标签

**显示内容**：
- 节点名称和类型
- 描述文本
- 推荐教科书列表
- 推荐课程列表
- 关联的关系（入度/出度）

**如何使用**：
1. 在右侧网状图中点击任何节点
2. 详情自动在左侧显示

### 3. 数据管理面板
**位置**：左侧边栏 → "数据" 标签

**功能**：
```
导出数据：
├─ 下载为 JSON 文件 (.json)
├─ 文件命名：knowledge-map-YYYY-MM-DD.json
└─ 用于备份或分享

导入数据：
├─ 粘贴 JSON 文本
├─ 验证格式
└─ 导入到当前视图

数据预览：
├─ 显示完整的 JSON 结构
├─ 支持复制到剪贴板
└─ 用于检查数据内容

本地保存：
└─ 自动保存到浏览器 localStorage
```

### 4. 网状图可视化
**位置**：右侧主区域

**交互方式**：
```
鼠标操作：
├─ 拖拽节点：移动到新位置
├─ 滚轮缩放：放大/缩小视图
├─ 右键拖拽：平移整个画布
└─ 点击节点：选中并显示详情

视觉特性：
├─ 蓝色方形：课程节点
├─ 金色菱形：概念节点
├─ 颜色边框：关系类型
│  ├─ 红色：前置课程
│  ├─ 绿色：后继课程
│  ├─ 浅绿：相关课程
│  └─ 黄色：应用
├─ 自动布局：物理引擎模拟
└─ 交互提示：悬停显示信息
```

## 📊 数据格式详解

### JSON 数据结构

```javascript
{
  "nodes": [
    {
      "id": "node_unique_id",        // 唯一标识符
      "label": "课程名称",           // 显示的名称
      "type": "course",               // 类型：course | concept
      "description": "详细描述",       // 节点描述
      "books": "教科书1\n教科书2",    // 多行教科书列表
      "courses": "课程1\n课程2",      // 多行推荐课程列表
      "shape": "box",                 // 形状：box | diamond
      "color": { ... },               // 颜色配置
      "font": { ... }                 // 字体配置
    }
  ],
  "edges": [
    {
      "id": "edge_unique_id",         // 唯一标识符
      "from": "source_node_id",       // 起始节点
      "to": "target_node_id",         // 目标节点
      "label": "前置",                // 边的标签
      "type": "prerequisite",         // 关系类型
      "arrows": "to",                 // 箭头方向
      "color": { ... },               // 颜色配置
      "font": { ... }                 // 字体配置
    }
  ]
}
```

## 🌐 GitHub 部署指南

### 前置条件
- GitHub 账户（https://github.com/signup）
- Git 已安装（或使用 GitHub Desktop）

### 部署步骤

**1. 创建仓库**
- 访问 https://github.com/new
- 仓库名：`knowledge-map`
- 选择 Public
- 创建

**2. 推送代码**
```bash
cd d:\code\knowledge-map

# 初始化（如果还没做）
git init
git add .
git commit -m "Initial commit"

# 连接到 GitHub（替换 YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/knowledge-map.git
git branch -M main
git push -u origin main
```

**3. 启用 GitHub Pages**
- 进入仓库 Settings
- 左侧选择 "Pages"
- Source: Branch `main` / Folder `/ (root)`
- 保存

**4. 完成！**
访问：`https://YOUR_USERNAME.github.io/knowledge-map/`

**5. 更新内容**
```bash
git add .
git commit -m "更新"
git push
# 1-2 分钟后网站自动更新
```

## ❓ 常见问题快速解答

**Q: 我不会 Git，怎么部署？**
A: 使用 GitHub Web 界面上传文件，或使用 GitHub Desktop。

**Q: 数据会丢失吗？**
A: 不会。有三层保障：
1. 本地存储（浏览器）
2. JSON 文件备份
3. GitHub 仓库

**Q: 能多人编辑吗？**
A: 目前是单用户本地编辑。可导出数据给他人导入。
  高级方案见 ADVANCED.md。

**Q: 手机上能用吗？**
A: 可以，访问 GitHub Pages URL 即可。
  但编辑体验更好的是电脑。

**Q: 能导出为图片？**
A: 可以，使用浏览器截图或打印为 PDF。
